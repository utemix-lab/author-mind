# Exports Model — архитектура смысловых экспортов

Этот документ описывает, как `author-mind` передаёт смыслы в другие репозитории  
через файлы в директории `exports/`.

Exports — это “выгрузки” смыслов: стиль, концепции, словари, принципы, модели мира.

---

## 1. Назначение Exports

Exports решают следующие задачи:

- передают в технические проекты:
  - стиль и тон повествования  
  - концептуальные рамки  
  - философские принципы  
  - worldbuilding  
- позволяют использовать author-mind как “генератор спецификаций смысла”  
- создают единый источник правды для:
  - тональности документации  
  - формата Stories  
  - лексики персонажей  
  - эстетики интерфейса

3. Формат export-файла
Расположение:

javascript
Копировать код
exports/export-XYZ-name.md
Рекомендуемый фронтматтер:

markdown
Копировать код
---
id: export-XYZ
type: export
title: "Название экспорта"
targets: []             # например: [vovaipetrova]
related_lines: []       # линии, из которых этот экспорт вырос
related_themes: []      # связанные темы
status: active
version: 1.0.0
---
Содержимое:

markdown
Копировать код
# export-XYZ — Название экспорта

## Purpose
(Зачем этот экспорт нужен, какие задачи он решает.)

## Content
(Основное содержание: стиль, принципы, правила, словарь, концепции.)

## Usage in Projects
(Как и где этот экспорт применяется в целевых репозиториях.)

## Notes for LLM
(Технические подсказки:
- что строго соблюдать
- что можно адаптировать
- как обновлять версию
)
4. Типы экспортов
Примеры типов:

style — тональность, голоса, речевые паттерны

concepts — базовые философские и сюжетные концепции

world — worldbuilding краткой формы

characters — набор характеристик персонажей

story-rules — правила построения Stories

visual-aesthetics — описательная база для визуального стиля

Тип может быть отражён в имени файла и/или в title.

5. Жизненный цикл экспорта
В author-mind созревают линии, темы, миры, эстетика.

Возникает необходимость применить это в конкретном проекте.

Создаётся export-файл:

из линий

из worlds / aesthetics / emotions

Export подключается в внешнем репо:

как документация (docs/style.md)

как источник для генераций (через LLM)

как конфиг смыслового уровня

По мере эволюции смыслов версия экспорта обновляется.

6. Связи с проектами
Каждый export обязательно связан с одним или несколькими проектами:

через поле targets во фронтматтере

через ссылки в projects/project-*.md

через использование в документации проекта

7. Требования к качеству Exports
чёткая цель (Purpose)

недвусмысленное содержание (Content)

explicit-описание применения (Usage)

аккуратные и стабильные формулировки (чтобы ИИ мог на них опираться)

отслеживаемая версия (version)

8. Назначение этого документа
Этот документ задаёт:

стандарт оформления экспортов

логическую архитектуру смыслового обмена между репозиториями

основу для создания “слоя смысла” в технических проектах

интерфейс между human-mind и machine-projects
