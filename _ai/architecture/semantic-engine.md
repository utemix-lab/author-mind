# Semantic Engine — проектирование движка смыслов

Semantic Engine (SE) — будущий вычислительный слой author-mind,  
который будет работать поверх онтологии, построенной в `ontology.md`.

Его задача — превратить author-mind из структуры хранения в **структуру жизни**,  
где смыслы могут:

- порождать друг друга,
- развиваться,
- соединяться,
- вытесняться,
- циркулировать между репозиториями,
- формировать новые линии и истории,
- и даже инициировать метаморфозы проекта.

---

# 1. Цель движка

Semantic Engine должен обеспечивать:

- **консистентность смыслового поля**  
- автоматическое обновление индексов  
- обнаружение связей между сущностями  
- генерацию новых мыслительных объектов  
- адаптацию author-mind под эволюцию проектов  
- машинное “понимание” структуры сознания автора  

Это не исполнитель кода.  
Это **вычислитель смыслов**.

---

# 2. Архитектурный принцип

Semantic Engine работает в трёх режимах:

## 2.1 Обработчик (Processor)
Превращает файлы → структуры → эмбеддинги → смысловые объекты.

## 2.2 Карту смысла (Sense Graph)
Создаёт граф смыслов:

idea ↔ line ↔ theme ↔ world ↔ story ↔ project ↔ export

Граф — не просто связи, а **веса**, **интенсивности**, **тяготения**.

## 2.3 Двигатель роста (Grower)
Инициирует:

- новые идеи,
- новые связи,
- расширения линий,
- предложения изменений,
- рекомендации по экспорту,
- контекстуальные доминанты.

Semantic Engine — это то, что делает систему **эволюционирующей**.

---

# 3. Основные компоненты

## 3.1 Parser
- читает фронтматтер  
- извлекает типы  
- фиксирует связи, темы, настроения  

## 3.2 Embedding Layer
- создаёт векторное представление каждой сущности  
- сравнивает, группирует  
- отслеживает “смысловые расстояния”

## 3.3 Relation Builder
- формирует рёбра графа смыслов  
- добавляет новые связи, если они присутствуют имплицитно  
- обнаруживает “тёмные области” — области без связей

## 3.4 Semantic Flow Engine
- определяет направление течения смыслов во времени  
- отслеживает развитие линий  
- выявляет зарождение новых  
- анализирует конфликты, дивергенцию, схлопывание смыслов  

## 3.5 Export Engine
- определяет, какие смыслы должны перейти в другие репозитории  
- предлагает стратегии связи  
- формирует “стиль проектов”

---

# 4. Модель работы

Файл появляется или изменяется

SE читает фронтматтер

SE конвертирует смысл в эмбеддинг

SE обновляет граф смыслов

SE предлагает:

новые связи

новые линии

расширения историй

потенциальные экспорты

архитектурные изменения

Автор подтверждает или отклоняет

Это “полуавтомат” —  
машина инициирует, автор направляет.

---

# 5. Что такое “смысловой объект” в SE?

Это сущность вида:

SemanticObject {
id
type
embedding
metadata
relations[]
}

**Это ядро будущей жизни системы.**

---

# 6. Эволюция движка

План развития:

1. Онтология — готово  
2. Semantic Engine — текущий файл  
3. Автоматическое построение графа смыслов  
4. Первичные эвристики роста  
5. Визуализация мышления  
6. “Жизненность” — когда система начинает порождать смысл без запроса
