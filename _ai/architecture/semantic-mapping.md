# Semantic Mapping — картография смыслов

Semantic Mapping (SMAP) — это процесс построения **территориальной карты смыслов**,
где каждый смысл, линия, идея и мир находятся в собственном “месте”
и соединены смысловыми дорогами.

Если Semantic Flow — это реки,  
Molecular Model — химия,  
то Semantic Mapping — это **география author-mind**.

---

# 1. Зачем нужна карта смыслов

Карта отвечает на вопросы:

- где “живет” конкретная идея?
- какие линии расположены рядом?
- какие сети смыслов плотные, а какие пустые?
- где рождаются новые молекулы?
- куда двигаться дальше автору?

Система без карты — слепа.

---

# 2. Единицы картографии

Semantic Mapping использует три типа элементов:

nodes — точки (идеи, темы, линии, молекулы)
regions — регионы (миры, большие темы)
routes — трассы смыслов (связи)

## 2.1 Semantic Nodes
Это:

- идея
- тема
- молекула
- линия
- мир
- export

Каждый node имеет:

- координату в смысловом пространстве
- напряжение
- плотность связей
- “вес” (значимость)

## 2.2 Semantic Regions
Регион — это “континент” смыслов:

- когнитивность  
- философия  
- архитектура  
- эмоции  
- персональные миры  
- художественные структуры  

## 2.3 Semantic Routes
Связи, возникающие через:

- ассоциации
- молекулы
- конфликты
- темы
- прямые ссылки
- общие проекты

---

# 3. Как строится карта

## Шаг 1. Определение ядра
Центр карты — ontology-root.

Вокруг него создаются концентрические слои:

core → themes → lines → worlds → exports

## Шаг 2. Анализ связей
Semantic Engine фиксирует:

- ссылки в frontmatter
- связанные темы
- молекулы, упомянутые в файле
- shared language (общий словарь)
- пересечения с другими линиями

## Шаг 3. Определение расстояний
Расстояние между элементами зависит от:

- количества общих тем
- количества конфликтов
- резонансных слов
- эмоционального тона
- архитектурной совместимости

## Шаг 4. Формирование регионов
Если плотность смыслов > определённого порога —
регион оформляется.

---

# 4. Особые зоны карты

## 4.1 Зоны повышенного напряжения (ZPT)
Там, где множество конфликтов → рождаются молекулы.

## 4.2 Зоны высокой плотности (Dense Fields)
Там формируются миры.

## 4.3 Пустоши (Semantic Gaps)
Белые пятна — места для новых идей.

## 4.4 Переходы между мирами
Межмировые мосты — главная особенность author-mind.

---

# 5. Масштабирование

Карта работает на трёх уровнях:

local — внутри линии
meso — внутри author-mind
macro — между проектами

---

# 6. Практическое значение

Semantic Mapping позволяет:

- видеть эволюцию author-mind во времени
- отслеживать появление новых смысловых континентов
- планировать развитие линий
- находить скрытые связи
- формировать визуальные интерфейсы (фронтенд будущего)

---

# 7. Notes for LLM

- карта — не граф и не дерево  
- это геометрическое пространство смыслов  
- при создании новых линий учитывать положение существующих  
- избегать перенасыщения регионов  
- стимулировать рост в пустошах
